get( )

get( )

Description

Get a single value that was stored

Available from Editor and Viewer, in all components, popups and modals.

Parameters

key String key of value to get.
Options (optional) Object Object that contains Wix.Data.SCOPE that indicates which scope to get the key from. Default scope is COMPONENT.
onSuccess (optional) function result will be the value attached to the key provided. For example: { key1: value1 }
onFailure (optional) function will be invoked when the provided key is not found

Examples

Availability

Since 1.61.0 and later.

Warning

This feature is experimental and it's behavior is not guaranteed. Not for use in production code.

getMulti( )

getMulti( )

Description

Get multiple values. To get multiple values, use the more efficient getMulti method. Pass it an array of keys and it will return a JSON object with those keys and matching values.

Available from Editor and Viewer, in all components, popups and modals.

Parameters

keys Array array of keys of values to get. All values returned will be from the provided scope.
Options (optional) Object Object that contains Wix.Data.SCOPE that indicates which scopes to get the keys from. Default scope is COMPONENT.
onSuccess (optional) function result will be a JSON Object with the key and value that were found. For example: { key1: value1 }
onFailure (optional) function invoked when the one of the keys is not found

Examples

Availability

Since 1.61.0 and later.

Warning

This feature is experimental and it's behavior is not guaranteed. Not for use in production code.

remove( )

remove( )

Description

Remove a single value. Future attempts to access this key will raise an exception until something is stored again for this key using one of the set methods.

Available in the settings panel, modal and popups (for example: the Manage window) in the Editor.

Parameters

key String key of value to remove.
Options (optional) Object Object that contains Wix.Data.SCOPE that indicates which scope to remove the key from. Default scope is COMPONENT.
onSuccess (optional) function result will be a JSON Object with the key and value that were found. For example: { key1: value1 }
onFailure (optional) function invoked when the provided key is not found

Examples

Availability

Since 1.61.0 and later.

Warning

This feature is experimental and it's behavior is not guaranteed. Not for use in production code.

set( )

set( )

Description

Stores the value under the key. If the key did not previously exist, it is created. If the key already exists, its existing value is overwritten with the new value. User can set: string, bool, number and json. Data is stored per component unless otherwise stated. Data will only be accessible from the component from which it was set. Data can also be stored globally, in which case it will be available across all components.

Available in the components, settings panel, modal and popups (for example: the Manage window) in the Editor.

Parameters

key String key of value to set
value Object | String | Number | Boolean value to set
Options (optional) Object Object that contains Wix.Data.SCOPE that indicates if data is accessible by specific comp or all apps comps installed
onSuccess (optional) function result will be a JSON Object mirroring the values given
onFailure (optional) function the function will return an object that specifies the error that occurred, it will be invoked in the following scenarios: 1. The app exceeded the provided 1K storage space. 2. Invalid value type, valid types are boolean, string, number and JSON.

Examples

Availability

Since 1.61.0 and later.

Warning

This feature is experimental and it's behavior is not guaranteed. Not for use in production code.
