<static> get(key, Options, onSuccess, onFailure)

Get a single value that was stored

Available from Editor and Viewer, in all components, popups and modals.

Parameters:
Name Type Argument Description
key String

key of value to get.
Options Object <optional>

Object that contains Wix.Data.SCOPE that indicates which scope to get the key from. Default scope is COMPONENT.
onSuccess function <optional>

result will be the value attached to the key provided. For example: { key1: value1 }
onFailure function <optional>

will be invoked when the provided key is not found
Since:
  • 1.61.0
Author:
  • mayah@wix.com
Experimental:
  • True
Source:
Data.js
Example
// returns a key named 'myKey' from the APP scope
Wix.Data.Public.get(‘myKey’, { scope: 'APP' }, function(result){
 console.log(result);
});

<static> getMulti(keys, Options, onSuccess, onFailure)

Get multiple values. To get multiple values, use the more efficient getMulti method. Pass it an array of keys and it will return a JSON object with those keys and matching values.

Available from Editor and Viewer, in all components, popups and modals.

Parameters:
Name Type Argument Description
keys Array

array of keys of values to get. All values returned will be from the provided scope.
Options Object <optional>

Object that contains Wix.Data.SCOPE that indicates which scopes to get the keys from. Default scope is COMPONENT.
onSuccess function <optional>

result will be a JSON Object with the key and value that were found. For example: { key1: value1 }
onFailure function <optional>

invoked when the one of the keys is not found
Since:
  • 1.61.0
Author:
  • mayah@wix.com
Experimental:
  • True
Source:
Data.js
Example
// Returns key1 and key2 from the APP scope
Wix.Data.Public.getMulti([‘key1’, ‘key2’], { scope: 'APP' }, function(results){
 for(key in results) {
     console.log(key + ‘: ‘ + results[key]);
 };
});

<static> set(key, value, Options, onSuccess, onFailure)

Stores the value under the key. If the key did not previously exist, it is created. If the key already exists, its existing value is overwritten with the new value. User can set: string, bool, number and json. Data is stored per component unless otherwise stated. Data will only be accessible from the component from which it was set. Data can also be stored globally, in which case it will be available across all components.

Available in the components, settings panel, modal and popups (for example: the Manage window) in the Editor.

Parameters:
Name Type Argument Description
key String

key of value to set
value Object | String | Number | Boolean

value to set
Options Object <optional>

Object that contains Wix.Data.SCOPE that indicates if data is accessible by specific comp or all apps comps installed
onSuccess function <optional>

result will be a JSON Object mirroring the values given
onFailure function <optional>

the function will return an object that specifies the error that occurred, it will be invoked in the following scenarios: 1. The app exceeded the provided 1K storage space. 2. Invalid value type, valid types are boolean, string, number and JSON.
Since:
  • 1.61.0
Author:
  • mayah@wix.com
Experimental:
  • True
Source:
Data.js
Example
// Will be accessible only from the component from which it was called
Wix.Data.Public.set(‘defaultSettings’,

    "showSuccessMsg": true,
    "formFields": {
        "name": "What is your name?",
        "quest": "What is your quest?",
        "color": "What is your favorite color?"
    }
  });
